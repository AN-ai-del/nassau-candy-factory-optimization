# 💻 Getting Started

## 1. Fork the Repository

Click the **Fork** button on GitHub.

---

## 2. Clone Your Fork

```bash
git clone https://github.com/<your-username>/nassau-candy-factory-optimization.git
```

---

## 3. Navigate to the Project

```bash
cd nassau-candy-factory-optimization
```

---

## 4. Create a Virtual Environment

### Windows

```bash
python -m venv .venv
.venv\Scripts\activate
```

### Linux/macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
```

---

## 5. Install Dependencies

```bash
pip install -r requirements.txt
```

Commit your work.

# 📝 Coding Standards

Please follow these guidelines.

## Python

- Follow PEP 8.
- Use meaningful variable names.
- Keep functions focused on one task.
- Add docstrings where appropriate.
- Avoid duplicate code.

---

## Project Structure

Maintain the existing modular architecture.

For example:

```text
src/
├── data/
├── models/
├── optimization/
├── utils/
└── visualization/
```

Avoid placing business logic directly inside the Streamlit dashboard.

---

## Documentation

When adding new features:

- Update the README if necessary.
- Update architecture documentation if the design changes.
- Add comments where helpful.
- Keep documentation synchronized with implementation.

---

# ✅ Commit Message Guidelines

Use clear and descriptive commit messages.

Examples:

```text
Add recommendation engine

Improve dashboard analytics

Refactor prediction module

Update project documentation

Fix feature engineering bug
```

Avoid vague commit messages such as:

```text
Update

Changes

Fix

Stuff
```

---

# 🔍 Pull Request Guidelines

Before opening a Pull Request:

- Ensure the project runs successfully.
- Verify there are no broken imports.
- Update documentation if required.
- Keep pull requests focused on a single feature or fix.
- Provide a clear description of your changes.

---

# 🐞 Reporting Issues

If you discover a bug, please include:

- Operating System
- Python version
- Error message
- Steps to reproduce
- Expected behavior
- Screenshots (if applicable)

This information helps reproduce and resolve issues more efficiently.
